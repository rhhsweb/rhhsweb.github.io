Head Element

The head element contains important metadata about the website. There should be no visible elements in this section. 

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <meta http-equiv="X-UA-Compatible" content="IE=edge">
        <meta name="description" content="This is the description for this website">
        <meta name="keywords" content="key, words, related, to, this, website">
        <meta http-equiv="refresh" content="5"> <!-- refresh the current page in 5 seconds -->
        <meta http-equiv="refresh" content="0;URL=newpage.html"> <!-- redirect to a new page -->
        <meta name="copyright" content="Copyright owner's name">
        <title>RHHS Web Development</title>
        <!-- icons -->
        <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
        <link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
    </head>
    <body>
        <!-- website content -->
    </body>
</html>

Meta Elements

There are many <meta> elements that a webpage can have. Most of them tend to have name that are fairly self-explanatory. Generally, the most important <meta> elements to have is the one that defines the source code character encoding, defines the website viewport, and enables X-UA-Compatible mode. 

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta http-equiv="X-UA-Compatible" content="IE=edge">

Source Code Character Encoding

The <meta charset="UTF-8"> element is used to tell the browser how the HTML file is encoded. Most of the time, this will be UTF-8.

Viewport

The viewport of a website is the screen area in which the website can be seen. This differs to the actual size of webpage, which can be larger using scrollbars.

On mobile devices, the viewport is rendered with a larger width than the screen, then is scaled down. For example, a screen with a width of 640px might have a virtual viewport of 900px. Many websites that are not optimized for mobile will break on smaller screens. A larger virtual viewport solves this issue.

If your website is designed for mobile devices (like this one!), then the <meta name="viewport"> can prevent mobile devices from using a larger viewport.

X-UA-Compatible Mode

Old Internet Explorer browsers often do different things than are expected by a regular browser. But, newer version of Internet Explorer tend to fix these issues. The <meta http-equiv="X-UA-Compatible" content="IE=edge"> element tells Internet Explorer browsers to use the highest mode possible to render websites.

For more information on the different <meta> elements, visit <meta>: The metadata element from MDN Web Docs.

Icons

Icons are the small image displayed in the browser tab, browser history, and bookmarks tab. Multiple sized icons are often added just in case one device cannot load one icon size. 

<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
